# 参与方式（User Journey）

1. **接任务**：在 Issues 里挑你感兴趣的 Quest
2. **做任务**：学习 / 开发 / 推广
3. **交任务**：
   - **代码类**：
     - Fork 仓库后，在 `submissions/` 下新建目录：`submissions/<issueId>-<githubId>-<slug>/`
     - 复制 `submissions/_template/README.md` 作为你的提交说明，并补齐运行/验收/产出链接
     - 提交 PR，并在 **PR 描述中**写：`Fixes #12`（用于自动发放积分）
   - **非代码类**：在 Issue 下评论截图/链接（文章链接、视频链接等）
4. **核销与计分**：
   - **代码类**：管理员 Review 后合并 PR；**PR 合并时会自动给 PR 作者发放积分**（按关联 Issue 的 `Points: XX`）
   - **非代码类**：由维护者在 Issue 下回复 `/award @你的用户名` 发放积分
