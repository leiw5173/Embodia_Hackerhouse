# Quest（任务）编写规范

每一个 Quest 都是一个 Issue。建议用 `.github/ISSUE_TEMPLATE/quest.yml` 表单创建。

## 必填要素

- **类型**：`Quest: Learning / Quest: Coding / Quest: Promotion`
- **分值**：`Points: 10 / 50 / 100`
- **验收标准**：必须可检查（链接、截图、PR、可复现步骤等）
- **提交方式**：清晰说明用 PR 还是评论链接

## 推荐结构（示例）

标题：`[新手任务] 跑通 Hello World 数字人`

Labels：`Quest: Coding` + `Points: 50` + `Status: Open`

内容：

- 背景/目标
- 验收标准
- 参考链接
- 提交方式（PR / 评论）
